¿Que es Eclipse?

Es una potente y completa plataforma de programación,desarrollo y compilación de elementos tan variados como sitios web,programas en C++ ó aplicaciones Java.No es mas que un entorno de desarrollo integrado(IDE) en el que encontrarás todas las herramientas y funciones necesarias para tu trabajo, recogidas además en una atractiva interfaz que lo hace fácil y agradable de usar. En otras palabras, Eclipse es un entorno de desarrollo integrado, de Código abierto y Multiplataforma.

Caracteristicas del IDE Eclipse

  • Dispone de un editor de texto con resaltado de sintaxis donde puedes ver el contenido del fichero en el que estás trabajando.
  • Contiene una lista de tareas y otros modulos similaras.
  • La compilación es en tiempo real.
  • Tiene pruebas unitarias con Junit.
  • Integración con Ant, asistentes(Wizards) para creación de proyectos, clases, tests,etc., y refactorización.

Instalacion del IDE Eclipse

Su instalación es muy sencilla:

  • Visitar http://www.eclipse.org y pulsar “Download Eclipse”
  • Descargar la versión actual “Eclipse IDE for Java Developers”
  • Descomprimir en la carpeta donde queramos instalalar Eclipse
  • Ejecutar el programa “eclipse” que hay dentro de esa carpeta

Creando un proyecto con contenido web

1- Seleccionamos en la pestaña de file en donde dice new Proyect y se abrirá una ventana nueva en la cual debemos de seleccionar si el tipo de proyecto que vamos a comenzar,en nuestro caso sera un proyecto web con contenido estático, ejemplo(HTML).



2- Debemos crear un repositorio en git, para ello haremos click derecho sobre la carpeta del proyecto (en eclipse), seleccionar “TEAM”y luego “SHARE PROJECT”, luego seleccionar “Use or create repository in parent folder of project”.



3- Seleccionar abajo a la izquierda en “Create Repository".

NOTA:

Se puede ver que eclipse no recomienda crear repositorios en la propia carpeta de trabajo “Workspace” por lo que podemos crear el repositorio en alguna otra carpeta previamente creada para este fin.



4- Para empezar a trabajar en un archivo, podemos desglosar las carpetas contenidas en el proyecto y sobre la carpeta “Web content” hacer click derecho y seleccionar el tipo de archivo en el que empezaremos a trabajar, ejemplo HTML.

Haciendo un push

1- Primero debemos crear un repositorio en nuestra cuenta de github y copiar la URL de dicho repositorio. Para hacer un push a nuestro repositorio en remoto, debemos hacer click derecho sobre el proyecto que queremos subir(eclipse) y click en remote, donde se abrirá una ventana en la cual debemos pegar la URL previamente obtenida de nuestro github, además de comprobar los datos con nuestra contraseña y cuenta github.



2- En esta ventana podemos seleccionar las ramas a las que queremos hacer un push. Seleccionar la rama y seleccionar +Add Spec.



3- Luego de hacer el push deberá aparecer este mensaje donde aparecerá la rama que ha sido subida y el repositorio al que ha sido subida.

Creando un archivo .gitignore

Siempre que comenzamos un proyecto se crean una serie de archivos(configuración,temporales,etc) que no nos interesan a la hora de hacer un commit, por lo que debemos de buscar una solución para ignorarlos. Para esto podemos agrupar todos este código en un solo archivo al que llamaremos .gitignore el cual podremos utilizar para pegarlo en cada proyecto nuevo y actualizar para que todos estos archivos se conviertan en uno solo(.gitignore).

Al crear este archivo en eclipse(git) se agruparan todos los archivos a ignorar en uno solo que llamaremos .gitignore.
